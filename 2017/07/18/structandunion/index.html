结构体和联合体

1.定义

struct

struct是一种复合数据类型，其构成元素既可以是基本数据类型（如int、long、float等）的变量，也可以是一些复合数据类型（如array、struct、union等）的数据单元。对于结构体，编译器会自动进行成员变量的内存对齐，以提高效率。
缺省情况下，编译器为结构体的每个成员按其自然对界条件分配空间。各个成员按照它们被声明的顺序在内存中顺序存储，第一个成员的地址和整个结构的地址相同。关于内存对齐可以参考：C++中的内存对齐

而C++中的struct相比于C中的struct有以下区别：

  • 在C中struct是一种用户自定义数据类型，而在C++中它是一种抽象数据类型
  • 在C中struct没有权限设置，而在C++中有访问权限
  • 在C中struct不能包含成员函数，不能继承，也不能实现多态，而这一切在C++中都可以
    struct在使用中需要注意以下几点：
  • 在用模版的时候只能写template < class Type >或template < typename Type >不能写template < struct Type >
  • struct可以相互嵌套使用，但是，struct可以包含struct指针而不能直接包含struct变量

C++中的struct使用起来与class十分相似，它与class主要有以下区别：

  • struct不能含有方法而class可以
  • struct默认成员访问权限，默认继承访问权限都为public，而class是private的

此外，struct关键字起初是为了和C兼容才出现在C++中的，并且struct与class在不涉及多态和继承的情况下，存取效率是完全相同的。

union

unnion，即联合体，是一种特殊的类，也是一种构造类型的数据结构。一个union内可以定义多种不同的数据类型，一个union对象可以装入该union中所定义的任何一种数据类型，并且这些数据共享同一段内存，以节省空间。这是union非常特殊的地方，此外，union变量所占用的内存长度等于最长的数据成员的内存长度。

union起源与C，它在C/C++中的使用并没有太大的差别，唯一需要注意的地方详见C++中struct与class的区别的最后一点。

C++中union与class的区别：

  • union不支持继承
  • union不能定义虚函数（也就是说无法实现多态）
  • union默认成员访问权限为public（这点与struct一样）
  • union中不允许存在静态和引用类型的成员

关于最后一点，解释一下：由于union内的成员共享内存，所以静态，引用之类的东西是不可能存在的；同时，不是所有类都可以成为union的成员变量，如果一个类（或者其父类）含有自定义的构造函数，析构函数，拷贝函数，复制拷贝运算符或虚函数中的任意一个，那么这个类存在共享内存的行为，编译器无法保证这些对象不被破坏，也无法保证正确的调用构造，析构函数，十分可能引起内存泄漏，所以，该类的对象无法成为union的成员。

上述问题出现的原因就是C语言中没有类的概念，而C++却引入了这一复杂的概念，所以在C++中使用union时应尽量保持C的风格，避免union与对象同时使用。

3.总结

struct在概念上更倾向于一种用户自定义的复杂数据结构，在使用上应与具有成员函数的class区分开来。就拿struct未定义构造函数时使用{}初始化举个例子，{}的方式是一种简单的参数列表进行初始化，适合于简单的数据结构，而如果struct加入了一个构造函数，那么这个struct就体现出了对象的特性，{}也将不再有效，这个现象本质上是由于默认访问控制权限造成的，{}初始化的方式仅在public的访问控制下可以使用。
那么如何区分一个类(class struct union)是否可以使用列表初始化来完成初始化工作呢？关键问题看这个类是否是一个聚合体（aggregate)，首先看下C++中关于类是否是一个聚合体的定义：

  • 无用户自定义构造函数。
  • 无私有或者受保护的非静态数据成员
  • 无基类
  • 无虚函数
  • 无{}和=直接初始化的非静态数据成员。

从上面的讨论可以看出，struct更适合看成是一个数据结构的实现体，class更适合看成是一个对象的实现体。所以优秀的程序员应该在实际编码过程中来决定什么时候用struct什么时候用class。

接下来说一下union，它实际上是开辟一段固定大小的内存（按照最大成员大小），然后程序员自由的存取，这样的做法虽然节省了部分的空间，但我觉得最大的优势还是在于变量的复用性，以及对变量进行结构对齐。union在操作系统底层的代码中用的比较多，因为它在内存共享布局上方便且直观，所以网络编程，协议分析，内核代码上有一些用到union都比较好懂，简化了设计。
